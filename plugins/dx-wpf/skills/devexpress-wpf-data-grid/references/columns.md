# Columns — DevExpress WPF Data Grid

This reference covers everything about columns: how to define them (XAML, auto-generation, Smart Columns, ViewModel-driven via `ColumnsSource`), how to bind them (`FieldName` vs `Binding` vs unbound), how to organize them (bands, multi-row bands, fixed columns), and how to customize their appearance and behavior. The same APIs apply to `TreeListControl` (uses `TreeListColumn` instead of `GridColumn`).

## When to Use This Reference

Use this when you need to:

- Define columns explicitly in XAML or generate them automatically
- Decide between `FieldName`, `Binding`, and unbound columns
- Enable Smart Columns Generation (data-annotation-driven editor picks)
- Group columns into **bands** (logical headers above a set of columns)
- Use multi-row bands or nested (child) bands
- Fix columns / bands to the left or right edge
- Define columns in a ViewModel via `ColumnsSource` + `ColumnGeneratorTemplate`
- Customize column headers, widths, visibility, position
- Handle the `AutoGeneratedColumns` event for post-generation customization

## Three Ways to Define Columns

### 1. Auto-Generation

Set `DataControlBase.AutoGenerateColumns` to one of:

| Value | Effect |
|---|---|
| `None` (default) | No auto-generation. Define columns explicitly. |
| `AddNew` | Generate a column for each data source property, merge with explicit columns. |
| `AddNewAndReplace` | Generate all columns; explicit XAML columns are replaced. |

```xaml
<dxg:GridControl ItemsSource="{Binding Orders}"
                 AutoGenerateColumns="AddNew"
                 EnableSmartColumnsGeneration="True"/>
```

Columns appear in data-source-property order. Collection properties are skipped.

After generation, the grid raises `DataControlBase.AutoGeneratedColumns` — handle it to fine-tune individual columns (hide some, set editors, assign formats):

```csharp
grid.AutoGeneratedColumns += (s, e) => {
    grid.Columns["InternalId"].Visible = false;
    grid.Columns["Price"].EditSettings = new TextEditSettings { DisplayFormat = "c2" };
};
```

Source: `articles/controls-and-libraries/data-grid/grid-view-data-layout/columns-and-card-fields/create-columns-and-bind-them-to-data-properties.md`.

### 2. Manual XAML

```xaml
<dxg:GridControl ItemsSource="{Binding Orders}">
    <dxg:GridControl.Columns>
        <dxg:GridColumn FieldName="ProductName"/>
        <dxg:GridColumn FieldName="UnitPrice"/>
    </dxg:GridControl.Columns>
</dxg:GridControl>
```

`GridControl.Columns` is the content property — you can omit `<dxg:GridControl.Columns>` and put `<dxg:GridColumn>` directly inside `<dxg:GridControl>`.

### 3. Code-Behind

```csharp
using DevExpress.Xpf.Grid;

var column = new GridColumn { FieldName = "ProductName" };
grid.Columns.Add(column);

// Bulk add:
grid.Columns.AddRange(new[] {
    new GridColumn { FieldName = "ProductName" },
    new GridColumn { FieldName = "UnitPrice" },
});

// Generate columns for all data source properties:
grid.PopulateColumns();   // Clears existing columns first
```

## Smart Columns Generation

`EnableSmartColumnsGeneration="True"` enables auto-selection of editors per data type AND uses `[Display]` data annotations to drive band groupings, captions, ordering, and visibility.

| Data Field Type | Generated Editor | Notes |
|---|---|---|
| `int`, `long`, `short`, `byte` (and nullable) | `TextEditSettings` with `MaskType="Numeric"` | |
| `decimal` (and nullable) | `TextEditSettings` with `MaskType="Simple"`, `Mask="C"` | Currency display |
| `bool`, `bool?` | `CheckEditSettings` | |
| `enum` | `ComboBoxEditSettings` with `EnumItemsSource` | |
| `DateTime`, `DateOnly` (and nullable) | `DateEditSettings` | |
| `DateTime` + `[DataType(DataType.DateTime)]` | `DateEditSettings` with `Mask="g"`, `MaskUseAsDisplayFormat="True"` | |
| Numeric + `[DataType(DataType.PhoneNumber)]` or `[Phone]` | `TextEditSettings` with `Mask="(000) 000-0000"`, `MaskUseAsDisplayFormat="True"` | |
| `string` | `TextEditSettings` | |

Data annotations that drive Smart Columns:

| Attribute | Effect on column |
|---|---|
| `[Display(Name = "...")]` | Sets `Column.Header` |
| `[Display(Order = N)]` | Sets `Column.VisibleIndex` |
| `[Display(GroupName = "...")]` | Generates a band (see § "Bands → Use Data Annotation Attributes") |
| `[Display(AutoGenerateField = false)]` | Hides the column |
| `[Display(AutoGenerateField = true)]` | Force generation (needed for non-string `IEnumerable`) |
| `[DisplayFormat(DataFormatString = "...")]` | Sets `EditSettings.DisplayFormat` |
| `[Editable(false)]` | Sets `ReadOnly = true` |
| `[Required]`, `[StringLength]`, `[Range]`, `[RegularExpression]`, `[CustomValidation]` | Hook into validation pipeline (see `references/editing.md`) |

Source: `articles/controls-and-libraries/data-grid/grid-view-data-layout/columns-and-card-fields/applying-data-annotations.md`.

## `FieldName` vs `Binding` vs Unbound

The Grid offers three ways to associate a column with data:

| Approach | Setup | Pros | Cons |
|---|---|---|---|
| **`FieldName`** | `<dxg:GridColumn FieldName="ProductName"/>` | Fastest (uses `PropertyDescriptor`). All grid features work. | Single property only. Limitations on dynamic types and nested paths. |
| **`Binding`** | `<dxg:GridColumn Binding="{Binding ProductName, Mode=TwoWay}"/>` | Full WPF binding (converter, mode, multi-bind). Works with `ExpandoObject`. | Slower for sort / filter (uses standard WPF binding). Read-only unless `Mode=TwoWay`. |
| **Unbound** | `<dxg:GridColumn FieldName="Total" UnboundDataType="{x:Type sys:Decimal}"/>` + handle `CustomUnboundColumnData` or use `CustomUnboundColumnDataCommand` | Display calculated values from any source. Editable if you implement `IsSetData`. | Code-driven (event or MVVM command). |

`FieldName` limitations (per the docs):

- Dynamic types are not supported (`ExpandoObject` IS supported — special case).
- Cannot access collection members (`FieldName="SomeItems[0]"`).
- Collection properties cannot be edited.
- Source items must all be the same type (or `object` collection, where the first item's type is used).
- Nested paths (`"Owner.Address.Street"`) — changes are not detected unless you set `DataControlBase.DetectNestedPropertyChanges="True"`.

`Binding` limitations:

- Sort / filter are slower (Binding pipeline vs PropertyDescriptor).
- Behaves like an unbound column for server mode / `ICollectionView` (see [data-binding.md](data-binding.md)).
- Read-only by default — set `Mode=TwoWay` to allow editing.

> **Cannot use both** `Binding` and `FieldName` simultaneously if they refer to an existing data-source property. Pick one. Combining `FieldName` (for the data path) with an unbound column requires using `Binding` instead.

Source: `articles/controls-and-libraries/data-grid/grid-view-data-layout/columns-and-card-fields/create-columns-and-bind-them-to-data-properties.md`.

## Unbound Columns

Columns whose value comes from code, not a data source property. Use for:

- Calculated values (`Total = UnitPrice * Quantity`)
- Display of data from a parallel collection (e.g., lookup by row index)
- Aggregations that depend on multiple source columns

### Setup

```xaml
<dxg:GridColumn FieldName="Total"
                UnboundDataType="{x:Type sys:Decimal}"/>
```

`UnboundDataType` is required — it controls sorting, filtering, default editor, and export type. Without it, the column is treated as `object`.

### Supply Values via Event

```csharp
private void grid_CustomUnboundColumnData(object sender, GridColumnDataEventArgs e) {
    if (e.IsGetData && e.Column.FieldName == "Total") {
        var product = (Product)e.Row;
        e.Value = product.UnitPrice * product.Quantity;
    }

    if (e.IsSetData && e.Column.FieldName == "Total") {
        // Save edited value back to a custom data source
        myDictionary[e.ListSourceRowIndex] = (decimal)e.Value;
    }
}
```

`e.IsGetData` fires when the grid asks for a value to display. `e.IsSetData` fires when the user edits the cell.

### Supply Values via MVVM Command

```xaml
<dxg:GridControl CustomUnboundColumnDataCommand="{Binding UnboundColumnDataCommand}">
    <dxg:GridColumn FieldName="Total" UnboundDataType="{x:Type sys:Decimal}"/>
</dxg:GridControl>
```

```csharp
using DevExpress.Mvvm.DataAnnotations;
using DevExpress.Mvvm.Xpf;

[Command]
public void UnboundColumnData(UnboundColumnRowArgs args) {
    if (args.IsGetData) {
        var item = (Product)args.Item;
        args.Value = item.UnitPrice * item.Quantity;
    }
}
```

`UnboundColumnRowArgs` is in `DevExpress.Mvvm.Xpf`. The `[Command]` attribute (from `DevExpress.Mvvm.DataAnnotations`) generates `UnboundColumnDataCommand` automatically.

Source: `articles/controls-and-libraries/data-grid/grid-view-data-layout/columns-and-card-fields/unbound-columns.md`.

## Bands (Grouped Column Headers)

A band is a logical group of columns with its own header. Use bands to visually organize related columns (e.g., "Product" | "Order Info" | "Pricing").

### XAML

```xaml
<dxg:GridControl ItemsSource="{Binding Source}">
    <dxg:GridControl.Bands>
        <dxg:GridControlBand Header="Product">
            <dxg:GridColumn FieldName="ProductName"/>
        </dxg:GridControlBand>

        <dxg:GridControlBand Header="Order Info">
            <dxg:GridColumn FieldName="Country"/>
            <dxg:GridColumn FieldName="City"/>
            <dxg:GridColumn FieldName="OrderDate"/>
        </dxg:GridControlBand>

        <dxg:GridControlBand Header="Pricing">
            <dxg:GridColumn FieldName="UnitPrice"/>
            <dxg:GridColumn FieldName="Quantity"/>
        </dxg:GridControlBand>
    </dxg:GridControl.Bands>
</dxg:GridControl>
```

> **You cannot use `GridControl.Columns` and `GridControl.Bands` at the same time.** Pick one.

### Code

```csharp
var pricing = new GridControlBand { Header = "Pricing" };
pricing.Columns.Add(new GridColumn { FieldName = "UnitPrice" });
pricing.Columns.Add(new GridColumn { FieldName = "Quantity" });
grid.Bands.Add(pricing);
```

### From Data Annotations

Apply `[Display(GroupName = "...")]` to source-class properties; the grid generates bands automatically when `EnableSmartColumnsGeneration="True"`:

```csharp
public class Product {
    [Display(GroupName = "Product")]    public string ProductName { get; set; }
    [Display(GroupName = "Order Info")] public string Country     { get; set; }
    [Display(GroupName = "Order Info")] public string City        { get; set; }
    [Display(GroupName = "Pricing")]    public double UnitPrice   { get; set; }
    [Display(GroupName = "Pricing")]    public int    Quantity    { get; set; }
}
```

Columns whose source property has no `GroupName` go into the first band.

### Nested Bands

```xaml
<dxg:GridControl.Bands>
    <dxg:GridControlBand Header="Personal Info">
        <dxg:GridControlBand.Bands>
            <dxg:GridControlBand Header="Name">
                <dxg:GridColumn FieldName="FirstName"/>
                <dxg:GridColumn FieldName="LastName"/>
            </dxg:GridControlBand>
            <dxg:GridControlBand Header="Birthday">
                <dxg:GridColumn FieldName="BirthDate"/>
            </dxg:GridControlBand>
        </dxg:GridControlBand.Bands>
    </dxg:GridControlBand>
</dxg:GridControl.Bands>
```

> A band cannot mix `Bands` and `Columns` in the same band (same rule as `GridControl`).

### Multi-Row Bands

Stack columns vertically within a band using `BandBase.GridRow` (attached) and `BaseColumn.VisibleIndex`:

```xaml
<dxg:GridControlBand Header="Performance">
    <dxg:GridColumn FieldName="MPGCity"      dxg:BandBase.GridRow="0" VisibleIndex="0"/>
    <dxg:GridColumn FieldName="MPGHighway"   dxg:BandBase.GridRow="0" VisibleIndex="1"/>
    <dxg:GridColumn FieldName="Transmission" dxg:BandBase.GridRow="1" VisibleIndex="0"/>
    <dxg:GridColumn FieldName="Gears"        dxg:BandBase.GridRow="1" VisibleIndex="1"/>
</dxg:GridControlBand>
```

Disable multi-row arrangement: `TableView.AllowBandMultiRow="False"`.

### Band Behavior Properties

| Property | Effect |
|---|---|
| `TableView.ShowBandsPanel` | Show / hide the bands panel above column headers |
| `TableView.BandSeparatorWidth` | Width of vertical lines between bands |
| `TableView.BandHeaderTemplate` | Custom `DataTemplate` for band headers (e.g., wrap text) |
| `TableView.BandHeaderToolTipTemplate` | Custom tooltip |
| `TableView.AllowBandMoving` | Users can drag bands |
| `TableView.AllowBandResizing` | Users can resize bands |
| `TableView.AllowChangeColumnParent` | Users can drag a column to another band |
| `TableView.AllowChangeBandParent` | Users can nest bands at runtime |
| `BandBase.OverlayHeaderByChildren` | Hide the band header, show children's headers in its place (useful for single-child bands) |

### Band Limitations

- `GridControl.Bands` and `GridControl.Columns` are mutually exclusive.
- A column inside a band cannot be **fixed** independently — only the entire band can be fixed.
- Multiple cell selection is not supported in banded views.
- A band's width is computed from its children — you can't set it explicitly.
- Cell Merging does not work in export if the grid has nested bands.

Source: `articles/controls-and-libraries/data-grid/grid-view-data-layout/bands.md`.

## Fixed Columns

Anchor columns or bands to the left or right edge so they don't scroll horizontally:

```xaml
<dxg:GridColumn FieldName="OrderId" Fixed="Left"/>
<dxg:GridColumn FieldName="Details" Fixed="Right"/>
```

| `FixedStyle` value | Effect |
|---|---|
| `None` (default) | Column scrolls with the rest |
| `Left` | Pinned to the left edge |
| `Right` | Pinned to the right edge |

To allow users to fix columns at runtime via the column-header context menu:

```xaml
<dxg:TableView AllowFixedColumnMenu="True"/>
```

Programmatic access to fixed/non-fixed column collections:

- `TableView.FixedLeftVisibleColumns`
- `TableView.FixedRightVisibleColumns`
- `TableView.FixedNoneVisibleColumns`

> **Fixed columns require auto-width to be disabled** (`TableView.AutoWidth="False"`) and the total column width to exceed the view's width. Otherwise horizontal scrolling is disabled and fixed columns have nothing to fix against.

Source: `articles/controls-and-libraries/data-grid/grid-view-data-layout/columns-and-card-fields/fixed-columns-and-bands.md`.

## MVVM: `ColumnsSource` and `BandsSource`

For ViewModel-driven column / band generation (when columns are not known at design time), use `Source` + `Generator` template pairs:

| Grid object | `Source` property | `Generator` template |
|---|---|---|
| Columns | `DataControlBase.ColumnsSource` | `DataControlBase.ColumnGeneratorTemplate` / `ColumnGeneratorTemplateSelector` |
| Bands | `DataControlBase.BandsSource` | `DataControlBase.BandGeneratorTemplate` / `BandGeneratorTemplateSelector` |
| Total Summaries | `DataControlBase.TotalSummarySource` | `DataControlBase.TotalSummaryGeneratorTemplate` |
| Group Summaries | `GridControl.GroupSummarySource` | `GridControl.GroupSummaryGeneratorTemplate` |
| Format Conditions | `TableView.FormatConditionsSource` | `TableView.FormatConditionGeneratorTemplate` |

```xaml
<dxg:GridControl ItemsSource="{Binding Data}"
                 ColumnsSource="{Binding ColumnDefinitions}">
    <dxg:GridControl.ColumnGeneratorTemplate>
        <DataTemplate>
            <dxg:GridColumn FieldName="{Binding FieldName}"
                            Header="{Binding HeaderText}"
                            Width="{Binding Width}"/>
        </DataTemplate>
    </dxg:GridControl.ColumnGeneratorTemplate>
</dxg:GridControl>
```

The ViewModel exposes `ColumnDefinitions` as `ObservableCollection<ColumnDefinition>` where `ColumnDefinition` is a POCO with `FieldName`, `HeaderText`, `Width`, etc. The grid generates a `GridColumn` per item using the template.

`ColumnGeneratorStyle` is also available for style-based setup (instead of full templates):

```xaml
<dxg:GridControl.ColumnGeneratorStyle>
    <Style TargetType="dxg:GridColumn">
        <Setter Property="HeaderStyle">
            <Setter.Value>
                <Style TargetType="dxg:BaseGridHeader">
                    <Setter Property="FontWeight" Value="SemiBold"/>
                </Style>
            </Setter.Value>
        </Setter>
    </Style>
</dxg:GridControl.ColumnGeneratorStyle>
```

Source: `articles/controls-and-libraries/data-grid/mvvm-enhancements/bind-to-collections-specified-in-the-viewmodel.md` and `mvvm-enhancements/examples/binding-to-a-collection-of-columns.md`.

## Per-Column Settings — API Surface

### `GridColumn` (`DevExpress.Xpf.Grid.GridColumn` : `ColumnBase`)

| Property | Type | Description |
|---|---|---|
| `FieldName` | `string` | Data source field name. |
| `Binding` | `BindingBase` | WPF binding (alternative to `FieldName`). |
| `Header` | `object` | Column caption. |
| `UnboundDataType` | `Type` | Type of an unbound column. Required for unbound. |
| `EditSettings` | `BaseEditSettings` | In-place editor config (see [cell-display-and-editing.md](cell-display-and-editing.md)). |
| `CellTemplate` / `CellDisplayTemplate` / `CellEditTemplate` | `DataTemplate` | Custom cell visual. See [cell-display-and-editing.md](cell-display-and-editing.md). |
| `Width` | `GridLength` | Fixed pixel width or star sizing. |
| `MinWidth` / `MaxWidth` | `double` | Resize limits. |
| `FixedWidth` | `bool` | Prevent resizing. |
| `Fixed` | `FixedStyle` | `None` / `Left` / `Right`. |
| `Visible` | `bool` | Hide column from view. |
| `VisibleIndex` | `int` | Column position (left-to-right). |
| `ShowInColumnChooser` | `bool` | Allow user to hide via Column Chooser. |
| `AllowEditing` | `bool?` | Override view-level edit permission. |
| `ReadOnly` | `bool` | Open editor for selection / copy, but no value change. |
| `IsReadOnlyBinding` | `BindingBase` | Per-row read-only state. |
| `AllowSorting` / `AllowGrouping` / `AllowFilter` | `bool?` | Per-feature permission overrides. |
| `SortMode` | `ColumnSortMode` | `Value` (default), `DisplayText`, `Custom`. |
| `SortOrder`, `SortIndex` | `ColumnSortOrder`, `int` | Initial sort. |
| `GroupIndex` | `int` | `-1` = not grouped; `0+` = group level. |
| `GroupInterval` | `ColumnGroupInterval` | `Default`, `Alphabetical`, `DateMonth`, `DateYear`, `DateRange`, `Value`. |
| `HeaderStyle` | `Style` | Target `dxg:BaseGridHeader` for theme-safe styling. |
| `EditFormVisible`, `EditFormVisibleIndex`, `EditFormCaption`, `EditFormStartNewRow`, `EditFormColumnSpan`, `EditFormRowSpan` | various | Edit Form layout (see [cell-display-and-editing.md](cell-display-and-editing.md)). |

### `GridControlBand`

| Property | Description |
|---|---|
| `Header` | Band caption |
| `Columns` | Child columns (cannot coexist with `Bands`) |
| `Bands` | Nested bands (cannot coexist with `Columns`) |
| `Fixed` | `FixedStyle.None/Left/Right` |
| `OverlayHeaderByChildren` | Hide band header, show children's headers in its place |
| `Visible` | Hide entire band |

## Customize Auto-Generated Columns

The simplest pattern — use `AutoGeneratedColumns` event:

```csharp
grid.AutoGeneratedColumns += (s, e) => {
    // Hide internal columns
    foreach (var col in grid.Columns)
        if (col.FieldName.StartsWith("_")) col.Visible = false;

    // Assign formats
    if (grid.Columns["Price"] is var price && price != null)
        price.EditSettings = new TextEditSettings { DisplayFormat = "c2" };

    // Reorder
    grid.Columns["Status"].VisibleIndex = 0;
};
```

The event fires once after auto-generation completes — operations here are cheaper than per-column XAML.

## Access Columns at Runtime

```csharp
GridColumn col = grid.Columns["FieldName"];        // by FieldName
GridColumn col = grid.Columns[0];                  // by index

// Iterate visible columns in display order:
foreach (var c in grid.View.VisibleColumns) { /* ... */ }

// Get the column at a specific cell location:
var col = grid.View.FocusedColumn;
```

`grid.Columns[...]` searches first by `FieldName`, then by `Name`. Use `GridColumn.Name` to give a column an explicit identifier when `FieldName` is empty (unbound case where no field exists).

Source: `articles/controls-and-libraries/data-grid/grid-view-data-layout/columns-and-card-fields/access-and-identify-columns.md`.

## End-User Column Operations

| Behavior | Property | Default |
|---|---|---|
| Reorder by drag | `TableView.AllowColumnMoving` | `True` |
| Resize by drag | `TableView.AllowResizing` | `True` |
| Auto-size to fit | `TableView.AutoWidth` | `False` (or `True` in Lesson 1 quickstart) |
| Best-fit on data change | `TableView.BestFitModeOnSourceChange` | `None` |
| Best-fit programmatically | `view.BestFit()` or `view.BestFitColumns()` | — |
| Show / hide via Column Chooser | `TableView.ShowColumnChooser` (or right-click header → "Column Chooser") | available |

## Apply to TreeListControl

All concepts in this reference map 1:1 to `TreeListControl`:

| GridControl | TreeListControl |
|---|---|
| `GridColumn` | `TreeListColumn` (same base class `ColumnBase`) |
| `GridControl.Columns` | `TreeListControl.Columns` |
| `GridControl.Bands` | `TreeListControl.Bands` (yes, TreeListControl supports bands too) |
| `GridControlBand` | `TreeListControlBand` (verify exact class — pattern is consistent) |
| `GridControl.AutoGeneratedColumns` event | `TreeListControl.AutoGeneratedColumns` |
| `GridControl.CustomUnboundColumnData` | `TreeListView.CustomUnboundColumnData` |
| `DataControlBase.ColumnsSource` | same (inherited) |
| `BaseColumn.Fixed` | same (inherited) |
